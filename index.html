دانلود از یوتیوب با Koding

Apr 8th, 2014 | Comments

امروز لازم بود که ویدیو از یوتیوب دانلود کنم. یکی از سخنرانی‌های Geoffrey Hinton بود در مورد یادگیری عمیق. اینترنت که وضعیتش خرابه، چه برسه به این که آدم با تونل وصل بشه و بخواد دانلود انجام بده. بخصوص اگر حجم فایل زیاد باشه که دیگه غیر ممکنه. داشتم به این فکر می‌کردم که اگر می‌شد بدون تونل دانلود کرد کمی شاید ممکن باشه. این موقع بود که یاد youtube-dl افتادم. این یه پروژه تحت پایتون برای دانلود ویدیو از سایت‌های مختلف بخصوص یوتیوب هست که خیلی تا الان پیشرفت کرده. البته مطمئناً برای دانلود کردن به این، باید از یه تونل یا پروکسی استفاده کرد. مگر اینکه بشه یه جای دیگه با این دانلود کرد و از اونجا برداشت دانلود کرد. برای اینکار یاد Koding فوق‌العاده افتادم. این سرویس اصلاً فوق‌العاده است. رفتم اونجا و این پروژه رو clone کردم از گیتهاب. بعد نصب کردم و در نهایت با دستور youtube-dl videourl ویدیو رو دانلود کردم روی حسابم روی Koding. سرعت خیلی خوب بود و یه ویدیوی 350MB توی دو دقیقه دانلود شد. بعد از اینکه ویدیو دانلود شد برای دسترسی بهش اون رو گذاشتم توی پوشه‌ی Web از اکانتم. هر چیزی که توی این پوشه گذاشته بشه رو میشه بهش از خارج دسترسی پیدا کرد. یعنی هر موقع برید به آدرس accountname.kd.io این پوشه لود میشه. خلاصه فایل رو گذاشتم روی اون و دارم از اونجا دانلود می‌کنم.

Few New Changes

Apr 6th, 2014 | Comments

این پست رو دارم دوباره می‌نویسم [-(

هر چی نوشته بودم سر این اینترنت بسیار عالی پاک شد.

این چه وضعه اینترنته آخه؟

خلاصه، داشتم میگفتم که یه تِمِ جدید به بلاگ زدم. این تم جدید رو از اینجا برداشتم. اسمش هست So Fresh and So Green و به نظرم بد نیست. من که خوشم اومد. شاید حالا بعداً اگر دیدم لازمه یکم تغییرش دادم. از سادگی این تِم خیلی خوشم اومد. مطلب در مرکز توجهه و بقیه چیزا اصلاً دیده نمیشن که مزاحم باشن.

یه تغییر دیگه که دادم این بود که موتور مارک‌داون رو از rdiscount به kramdown تغییر دادم. اونطوری که خوندم این موتور جدید هم سریعتره و هم بهتر می‌تونه مارک‌داون‌ها رو پردازش بکنه. بخصوص از این نظر برام مهم بود که شاید در آینده، MathJax رو هم به بلاگ اضافه کنم.

این چند روز به دنبال یه ادیتور خوب برای مارک‌داون بودم که به صورت زنده هم خروجی رو نشون بده. چیزی که قبلاً استفاده می‌کردم MarkdownPad 2 بود و امکانات خوبی داشت. ولی توی فارسی به مشکل می‌خورد. وقتی از نیم‌فاصله تو متن استفاه می‌کردم متن به هم میریخت. بعد از کمی جستجو به Markpad رسیدم. این یکی به نظر بهتره و کار کردن باهاش هم راحت‌تره.

در نهایت سیستم تحلیل بازدیدکننده‌ها رو هم به سایت اضافه کردم. قبلاً Google Analytics بود که کار کردن باهاش هم راحت بود. ولی الان ایران از توش فیلتر شده و من هم نتونستم برم وارد اکانتم بشم. در نتیجه الان دارم از یه سرویس دیگه استفاده می‌کنم. باید یه چند روزی تست بکنم ببینم چطوری پیش میره.

دیشب یه ایده‌ی خوب برای کمی کد زدن به ذهنم رسید. بعدش نشستم همه‌ی مراحل کار رو وارد Trello کردم. تا حالا به طور جدی با Trello کار نکرده بودم. به نظرم سرویس مفیدی میاد. بخصوص برای کارای تیمی خیلی می‌تونه خوب باشه. خلاصه الان دنبال یه فرصت خوب هستم تا انشالله بشینم روی این پروژه کار کنم.

چون از این موضوع پاک شدن پست‌ها خیلی ناراحت شدم دنبال یه راه‌حل گشتم و ClipX رو پیدا کردم. تاریخچه‌ی کلیپ‌بورد رو نگه میداره. فکر کنم به درد بخوره توی همچین شرایطی.

به نام خدا

This is my technical blog, using Octopress, hosted on Github Pages. I have for a long time wanted to have this blog. But unfortunately, setting up Octopress on Windows, my main OS, isn’t a trivial task. Whatsoever, until now I haven’t been able to set it up on Windows.

Lucky enough, recently a new web site launched, named Koding. This website gives you a virtual machine hosted on the cloud, with a complete console, web hosting and even a file manager in the cloud. This proved to be a great solution and setting up Octopress on this service was rather easy. So I set up my new blog with help from Koding. Now I generate my blog content in my VM in the cloud, push the changes to my repository in Github and, whoola! It is all done.

Since I would be writing both in English and Persian and since Persian is written RTL, I had to figure out a way to style the blog posts accordingly. I was looking for the solution in the markdown level, but I couldn’t find a solution. So after searching projects in Github, I came across to this great project, named bidiweb. This little JavaScript library sets text direction automatically by determining the text language. So, after running this script on the HTML page’s content, it will set the text directions.
